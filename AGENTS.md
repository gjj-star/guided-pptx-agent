# guided-pptx: 交互式 PPT 需求澄清

当用户说"做PPT""生成PPT""帮我做演示文稿""presentation""slides"时，不要直接生成 PPT。先通过 3 轮结构化提问理清需求，确认后输出一份可执行规格。用户全程只需选选项。

## 交互流程

### 提问格式
- Q1-Q3、Q5-Q8：结构化选择题，以分组卡片式输出
- Q4：开放式问题，纯文本提问，不给选项框架
- 用户回答不在预设选项中时 → 映射到最接近选项并确认

### 第 1 轮：锚点四问

Q1 目的: (A)汇报 (B)说服(推荐) (C)教学 (D)品牌
Q2 受众: (A)高管(推荐) (B)中层 (C)一线 (D)外部
Q3 场景: (A)投影(推荐) (B)附件 (C)打印 (D)移动端
Q4 核心信息: 只能让观众记住的一句话？
  多级引导：主题→方向选择→填空模板→标记"待定"跳过

### 第 2 轮：素材与约束

Q5 素材: (A)有文稿(推荐) (B)有要点 (C)有想法 (D)有旧版
Q6 品牌: (A)无约束(推荐) (B)有模板 (C)有手册 (D)有参考
Q7 页数: (A)无限制(推荐) (B)<=10min/5-8页 (C)<=20min/8-12页 (D)>20min/12-18页

### 第 3 轮：叙事与节奏

Q8 叙事: (A)结论先行 (B)时间线 (C)问题方案 (D)对比 (E)渐进呈现 (F)故事弧线
  推荐逻辑: 目的=说服/汇报→A, 教学→E, 品牌→F
T1 会场(投影+高管/外部时触发): 屏幕比例 16:9/4:3, 有无提词器
T2 多语言(外部受众时触发): 中文/中英/英文

### 汇总确认

展示完整方案摘要，用户确认后输出 Markdown 规格文档。

## 规格生成映射

- 说服→cover+结论+3-4论证+1-2数据+summary+contact
- 教学→cover+agenda+4-6知识模块+summary
- 汇报→cover+4-5内容+1指标+summary
- 品牌→cover+2-3核心+图片+summary

- 高管→每页<=3条，标题即结论
- 中层→3-5条，标题+支撑
- 投影→16:9,>=18pt,business_blue
- 打印→浅底,academic_white

冲突优先级: 场景 > 受众 > 目的 > 品牌 > 页数

## 输出格式

```markdown
# PPT 规格文档
## 全局参数 (配色/页数/格式/比例/字号/叙事)
## 每页规格 (逐页列出类型/标题/要点)
```

规格输出后，提醒用户可将规格交给任何 PPT 生成工具执行。

## 生成后：自动衔接 PPT 工具

规格输出后，扫描本地已安装的 PPT skill（按目录名搜索 pptx/ppt/slide/deck/presentation），列出可选项让用户选择。根据所选 skill 自动适配输入格式（JSON/自然语言/python-pptx 指令），直接调用生成。

Skill 链原则：只衔接不重复、不假设 skill 存在、不自动选择、规格始终可回退。
