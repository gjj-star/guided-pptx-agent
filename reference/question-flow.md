# 交互式问题流

每一步对应一次 AskUserQuestion 调用（≤4 个问题，每题 2-4 个选项 + 推荐默认值）。

---

## Batch 1：锚点四问（必问，不可跳过）

### Q1: 核心目的是什么？

| 选项 | 推荐 | 标签 |
|------|:----:|------|
| 汇报进展或同步信息 | | 汇报 |
| 说服决策者（批准预算/通过方案） | **(推荐)** | 说服 |
| 培训或教学 | | 教学 |
| 品牌展示或对外宣传 | | 品牌 |

**推荐理由**：大多数商务 PPT 的目的是推进决策而非单纯传递信息。"汇报"型 PPT 往往也需要隐含的说服力——你需要让上级认可你的进展。

**默认方案影响**：选择"说服"→推荐结论先行结构 + 数据支撑 + 明确 CTA 页。

### Q2: 受众是谁？

| 选项 | 推荐 | 标签 |
|------|:----:|------|
| 高层决策者（C-suite/VP/董事会） | **(推荐)** | 高管 |
| 中层管理者或项目负责人 | | 中层 |
| 一线团队或执行者 | | 一线 |
| 外部客户或合作伙伴 | | 外部 |

**推荐理由**：多数 PPT 的最终受众是决策层。即使给中层看，他们也需要向上汇报——按高管标准设计，两者都适用。

**默认方案影响**：选择"高管"→结论先行、低术语密度、每页≤1 个核心数字、字号≥20pt。

### Q3: 最终交付场景？

| 选项 | 推荐 | 标签 |
|------|:----:|------|
| 会议室投影或大屏 | **(推荐)** | 投影 |
| 邮件或微信发送，对方自行阅读 | | 附件 |
| 需要打印成纸质材料 | | 打印 |
| 手机或平板浏览 | | 移动端 |

**推荐理由**：会议室投影是最高频的商务场景。投影友好的设计（大字、高对比度、少字多图）对附件阅读也适用，反之则不成立。

**默认方案影响**：选择"投影"→16:9、字号≥18pt、深蓝/白配色、避免细线条。

### Q4: 只能让观众记住的一件事是什么？

这是一个开放式问题。如果用户回答不出，提示：

> "比如：'我们的 AI 方案能在 6 个月内将客服成本降低 40%'，或者 '团队 Q2 进度超出预期，只需要补充一个人即可按期交付'。用一个完整的句子说出来。"

**推荐理由**：这是整份 PPT 的脊椎。用户说不出时不应跳过——建议先花 2 分钟想清楚再继续。

**默认方案影响**：核心信息直接成为 cover 页和 summary 页的锚定文案。

## Batch 1 之后的条件分支

```
Q1 目的 → 决定默认叙事结构
  ├── "说服" → 推荐"结论先行"
  ├── "汇报" → 推荐"结论先行"（向上汇报其实也是说服）
  ├── "教学" → 推荐"渐进呈现"（从简单到复杂）
  └── "品牌" → 推荐"故事弧线"（情感连接优先）

Q2 受众 → 决定信息密度默认值
  ├── "高管" → 极简、结论先行、每页1核心信息
  ├── "中层" → 中等细节、可执行洞察
  ├── "一线" → 高细节、操作指引
  └── "外部" → 无内部术语、更多背景铺垫

Q3 场景 → 决定视觉参数默认值
  ├── "投影" → 16:9、深蓝+白、字号≥18pt、无细线
  ├── "附件" → 自解释、信息密度可更高、PDF输出
  ├── "打印" → 浅底、避免大面积深色块、注意黑白兼容
  └── "移动端" → 极简单页、文字精简到极限
```

## Batch 2：素材与约束三问

### Q5: 内容素材现在是什么状态？

| 选项 | 推荐 | 标签 |
|------|:----:|------|
| 已有完整文稿或逐页大纲 | **(推荐)** | 有文稿 |
| 只有关键数据和要点，需要协助组织 | | 有要点 |
| 只有初步想法，需要从零梳理 | | 有想法 |
| 已有旧版 PPT，需要翻新或参考 | | 有旧版 |

**推荐理由**：多数用户至少有几页笔记或数据。已在 Batch 1 中引导用户想清楚核心信息（Q4），这是素材梳理的第一步。

**默认方案影响**：选择"有要点"→需要先做大纲确认，再开始设计；选择"有想法"→需要多一轮大纲共创。

### Q6: 品牌视觉方面有什么约束？

| 选项 | 推荐 | 标签 |
|------|:----:|------|
| 无约束，可以自由设计 | **(推荐)** | 无约束 |
| 有公司模板，但可以灵活调整 | | 有模板 |
| 有严格品牌手册，必须遵守 | | 有手册 |
| 有参考 PPT 觉得方向对的——可提供 | | 有参考 |

**推荐理由**：多数场景下没有硬性品牌约束。即使有模板，也需要评估可用性后再决定是直接用还是调整为参考框架。

**默认方案影响**："无约束"→推荐 business_blue 配色（通用性最强）；"有模板"→先要过来评估；"有手册"→提取关键色彩和字体后适配。

### Q7: 页数和时长有限制吗？

| 选项 | 推荐 | 标签 |
|------|:----:|------|
| 无硬性限制，你根据内容推荐 | **(推荐)** | 无限制 |
| 10 分钟以内（建议 5-8 页） | | ≤10min |
| 10-20 分钟（建议 8-12 页） | | ≤20min |
| 20 分钟以上（建议 12-18 页） | | >20min |

**推荐理由**：没有特别要求时，按 10-15 分钟 / 8-12 页的行业标准节奏设计。这是最通用的框架，后续可根据素材量调整。

**默认方案影响**：选择"无限制"→默认 10 页 + 按每页 2-3 分钟演讲时间推算。

## Batch 3：叙事与节奏（1-2 问 + 场景触发）

### Q8: 偏好什么叙事顺序？

| 选项 | 推荐条件 | 标签 |
|------|----------|------|
| 结论先行（先抛结论，再展开论证） | Q1=说服/汇报时推荐 | 结论先行 |
| 时间线叙事（按时间顺序讲述发展） | Q1=汇报时可选 | 时间线 |
| 问题→方案→验证（经典提案结构） | Q1=说服时可选 | 问题方案 |
| 对比分析（方案 A vs B vs C） | 多方案比选时 | 对比 |
| 渐进呈现（从基础到复杂，逐步建构） | Q1=教学时推荐 | 渐进呈现 |
| 故事弧线（现状→理想→跨越，情感驱动） | Q1=品牌时推荐 | 故事弧线 |

**推荐理由**：根据 Batch 1 的答案动态推荐——
- 如果 Q1="说服/汇报"→推荐"结论先行"（高管没有耐心听铺垫）
- 如果 Q1="教学"→推荐"渐进呈现"（认知负荷理论：从简单到复杂）
- 如果 Q1="品牌"→推荐"故事弧线"（Duarte 框架：情感连接优先于信息传递）

**默认方案影响**：叙事结构决定页面组织顺序和 transition 设计。

### 场景触发：T1 大型演讲

触发条件：Q3="投影" 且 Q2="高管"或"外部"

设计理由：原始专家清单中 T1 触发于 C1=融资路演 或 C3=大会场。在压缩过程中，"大会场"被合并到"投影"场景以简化用户选项（避免场景过多）。T1 改用"高管/外部受众 + 投影"作为大会场的代理信号——这类受众通常意味着更重要的演示场合，可能需要大型场地。

| 问 | 选项 | 默认 |
|----|------|:--:|
| 会场条件？（屏幕比例、灯光情况） | 16:9 标准屏 / 4:3 老式屏 / 不确定 | 16:9 |
| 有无提词器或双屏？ | 有提词器 / 无（仅投影）/ 不确定 | 仅投影 |

**理由**：16:9 vs 4:3 是全局参数，事后改等于重做。提词器/双屏影响 speaker notes 策略。强光环境需要深色背景+高对比度。

### 场景触发：T2 多语言

触发条件：Q2="外部" 且 受众包含非中文母语者

| 问 | 选项 | 默认 |
|----|------|:--:|
| 需要中英双语或纯英文版吗？ | 仅中文 / 中英双语 / 纯英文 | 仅中文 |

**理由**：双语排版需要预留至少 30% 的额外文本空间，事后翻译会破坏所有版面。

## 汇总确认

所有 Batch 完成后，展示完整方案供用户确认：

```
根据你的回答，推荐方案如下：

目的：说服决策者
受众：高层（C-suite/VP）
场景：会议室投影
核心信息：[用户输入的内容]
素材状态：有要点，需协助组织
品牌约束：无约束，自由设计
页数：默认 10 页
叙事结构：结论先行
配色方案：business_blue
预计输出：output.pptx（16:9 PPTX 格式）

是否接受这套方案？或需要调整哪些设置？
```

如果用户选择调整，回到对应的 Batch 重新确认那一项（不需要全部重来）。
