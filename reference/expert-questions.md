# PPT 需求澄清专家问题清单

从企业汇报实践（McKinsey/Apple 级 presentation designer）和演示设计理论（Duarte/Reynolds/Tufte）双视角交叉提取。用于 guided-pptx skill 在生成前向用户进行结构化需求澄清。

---

## 第一优先级：必须问的

不确定这些就无法确定设计方向。

### 1. 这个 PPT 的核心目的是什么？
说服决策 / 汇报进展 / 培训教学 / 品牌展示 / 融资路演 / 学术分享？

**理由**：目的决定设计逻辑。同样的数据，路演和内部汇报的叙述语言、信息密度和视觉策略完全不同。这是所有后续决策的锚点。

### 2. 受众是谁？（具体身份级别和规模）
- 董事会 / C-suite（需要结论先行，低细节，高战略视角）
- 中层管理（需要可执行的洞察，中等细节）
- 一线团队（需要操作指引，高细节）
- 外部客户（需要建立信任，中等细节，避免内部术语）
- 混合受众（需要分层设计，关键页兼顾多重视角）
- 听众规模：小会议室（<10人）/ 中型（10-50人）/ 大会场（>50人）

**理由**：同一组数据和结论，给 CEO 看和给分析师看是两套语言体系。受众决定了信息密度、术语边界和叙述高度。

### 3. 最终交付场景是什么？
- 会议室投影（默认 16:9，字号需 ≥18pt，避免细线条）
- 大会场屏幕（需要高对比度，深色背景优先，极简设计）
- 邮件/微信附件自助阅读（需要自解释能力，信息密度可更高）
- 手机浏览（字号需更大，单页内容极少）
- 打印成册（避免深色背景，需要高对比度，注意彩打/黑白）

**理由**：场景决定字号下限、图表复杂度和色彩策略。投影场景下精美的细节可能完全看不见。

### 4. 如果只能让观众记住一件事，那件事是什么？（核心信息/Key Message）

**理由**：这是整份 PPT 的脊椎，每一页都应该服务于这个核心信息。如果客户说不出来，说明内容还没准备好。McKinsey 的经典原则：写完 PPT 后删除每一页，问自己"没有这页，核心信息还成立吗？"

### 5. 内容素材目前的状态是什么？
- 已有完整文稿/大纲（可直接进行视觉化）
- 只有数据和要点（需要协助梳理叙事结构）
- 只有口头想法/关键词（需要从零梳理，工作量 3-5 倍）
- 已有旧版 PPT 需要翻新

**理由**：素材成熟度直接决定工作量。只有口述需求的 PPT 和已有逐页大纲的 PPT，设计工作量差距巨大。

### 6. 有没有品牌视觉规范必须遵守？
- 有品牌手册（配色/字体/logo 使用指南）——请提供
- 有公司模板（需评估可用性）
- 无品牌约束（可自行设计）
- 有参考案例/竞品 PPT 觉得方向对的——请提供

**理由**：违反品牌规范在大型企业可能直接导致合规返工。如果有约束但设计师不知道，后期修改成本极高。

## 第二优先级：建议问的

答案会显著影响设计方向，但紧急程度稍低。

### 7. 你希望观众在演示结束后采取什么行动？
批准预算 / 签署合同 / 调整策略 / 转发分享 / 学习掌握 / 投票支持？

**理由**：好的商业演示要驱动行动。知道终点，才能设计路径。这决定了最后一页的 CTA 设计和整份 PPT 的情感弧线。

### 8. 演示者是谁？演讲经验如何？
- 是你本人，经验丰富——可以极简设计，每页只需标题
- 是你本人，不太习惯演讲——需要更多导航线索和文字支撑
- 是 CEO/高管——内容需提炼到精华，他们只需要提示
- 是团队成员——需要更详细的 speaker notes
- 不需要演讲，观众自行阅读（自解释模式）

**理由**：演示者的风格和能力直接影响每页的信息量。CEO 可能一个词撑起一页，不习惯演讲的人需要更多结构化提示。

### 9. 页数和时长有无限制？
- 总页数上限
- 演讲时长
- 是否需要预留 Q&A 时间
- 典型节奏：每页 2-3 分钟演讲时间

**理由**：页数和时长的关系决定了内容颗粒度。时长 20 分钟却无页数限制时，需要主动建议结构精简。

### 10. 偏好什么样的叙事结构？
- 结论先行 / 金字塔原理（McKinsey 式）——先抛结论，再展开论证
- 时间线叙事——按时间顺序讲述发展历程
- 问题→方案→验证——适合解决方案类提案
- 对比分析——适合竞品分析或方案比选
- 故事弧线（Duarte 式：现状 → 理想状态 → 如何跨越）

**理由**：叙事结构应该在排版前确定。McKinsey 式和 Apple 式是两套完全不同的设计语言和控制节奏的方式。

### 11. 数据敏感度和合规要求？
- 是否包含未公开的财务数据？
- 是否需要标注数据来源和引用？
- 是否有合规披露要求（上市公司/监管行业）？
- 内容是否在保密期内（竞标材料等）？

**理由**：在素材准备阶段就必须确认。因敏感信息导致整版重做是常见的灾难性返工场景。

### 12. 需要什么交付格式？
- PPTX（Windows PowerPoint，最通用）
- PDF（适合分发，不可编辑）
- Keynote（Mac）
- Google Slides（协作友好）
- 是否需要 speaker notes 页？
- 是否需要长期复用的模板文件？

**理由**：格式转换可能引入大量排版问题。Cross-platform（PPTX ↔ Keynote）的字体、动画兼容性是经典陷阱。

## 第三优先级：视情况问的

取决于项目类型和复杂度的补充问题。

### 13. 【外部数据/学术场景】需要引用第三方数据或研究报告吗？
需要的话，需在设计阶段预留来源标注空间。

### 14. 【多语言场景】有双语或多语言版本需求吗？
中英双语排版逻辑完全不同（字号、行距、文本框宽度）。事后翻译嵌入几乎必然破坏版面。

### 15. 【大型演讲场景】会场条件如何？
- 屏幕比例：16:9（标准）还是 4:3（老旧设备）？——事后改比例等于重做
- 舞台灯光：强光环境需要高对比度，深色背景优先
- 是否有提词器/双屏？——影响 speaker notes 的需求

### 16. 【团队协作场景】是否需要多人协作编辑？
需要预先约定命名规范、修改标记方式和版本管理流程。

### 17. 【印刷场景】是否有打印需求？彩色还是黑白？
打印友好的设计需要避免大面积深色背景、极细线条。黑白打印下信息分层不能依赖色彩。

### 18. 【数据可视化场景】图表数据源会持续更新吗？
如果图表需要定期更新数据，需要预留数据刷新后的排版容错空间。

### 19. 【动画/视频场景】是否有嵌入视频或复杂动画需求？
跨平台动画兼容性是经典陷阱。嵌入视频显著增大文件体积，需早期验证。

## 交叉验证：设计理论视角的独特贡献

企业汇报视角（agent 1）提供了实践的完整性，设计理论视角（agent 2）补充了以下企业视角容易遗漏的维度：

| 理论贡献 | 对应问题 | 企业视角是否覆盖 |
|----------|----------|:---:|
| Duarte 叙事弧线："现状→理想状态"的对比张力 | Q10 故事弧线选项 | 部分 |
| 开场方式偏好（数据/故事/设问/直接结论） | Q10 隐含 | 否——这是新增的细化维度 |
| 色彩禁忌与文化敏感度 | Q3/Q6 隐含 | 否——企业视角只关注品牌规范，未涉及文化禁忌 |
| 色盲友好设计（约 8% 男性受众） | Q3 隐含 | 否——可访问性是纯理论视角的贡献 |
| WCAG 2.1 无障碍标准合规 | 未纳入 | 仅在政府和教育场景相关，不纳入通用问题 |
| 图表跨页一致性（"数据配色字典"） | Q11/Q18 隐含 | 部分 |
| 互动环节设计（投票/讨论/QR码） | Q3/Q7 隐含 | 否——企业视角关注的是演讲后的行动，非演讲中的互动 |
| 多版本输出（演讲版/阅读版/社交媒体版） | Q12 交付格式 | 部分 |
| 动画策略（无/渐进呈现/叙事性动效） | Q19 视情况 | 覆盖 |
| 情绪节奏设计（像配乐一样引导情绪起伏） | Q10 隐含 | 否——高预算场景的价值增量 |
| 非标准图表类型（桑基图/雷达图/树图） | Q18 隐含 | 部分 |

### 纳入策略

以下设计理论视角的独特贡献将被纳入最少必要问题集：

1. **色彩禁忌**——合并到品牌规范问题中（Q6），增加"有无颜色使用禁忌"追问
2. **可访问性**——合并到交付场景问题中（Q3），增加"是否有色觉障碍受众"提示
3. **开场偏好**——合并到叙事结构问题中（Q10），增加开场方式子选项
4. **互动环节**——合并到目的/场景问题中（Q1/Q3），作为场景的一个维度
5. **情绪节奏**——作为"视情况问的"保留，高预算/高影响力场景触发

以下设计理论视角的贡献**不纳入**通用问题集（原因）：

- WCAG 合规：政府和教育场景占比极低，由场景触发追加问题
- 多版本输出：大多数场景只需一个版本，由交付格式问题触发追问
- 非标准图表：内容素材阶段自然暴露，无需单独提问
